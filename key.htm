XSLT key()

Photo: benisntfunny - Sunset

The XML

				<data>
					<item region="Italy">
							<name>Guiseppe</name>
					<item>
					<item region="Germany">
							<name>Hans</name>
					<item>
				 ...
				<data>

Regular XPath

				<xsl:template match="/">
					<xsl:for-each select="data/item[@region='Germany']">
							<xsl:value-of select="name" />
					</xsl:for-each>
				</xsl:template>

Using key()

				<xsl:key name="regions" match="data/item" use="@region" />
				 ...
				<xsl:template match="/">
					<xsl:for-each select="key('regions','Germany')">
							<xsl:value-of select="name" />
					</xsl:for-each>
				</xsl:template>

Results

Advantages

  • Allows the XSLT processor to define a search key on your data
  • Improves performance dramatically

Disadvantages

  • Only very strict, simple Xpath expressions possible
